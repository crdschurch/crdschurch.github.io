Heads Up:This project is currently on hold. We've chosen a new development vendor and we're currently in process of establishing the contract and project plan.

Project Dashboard

The homepage for project "Milacron". Here you'll find links to key documentation, milestones, and helpful information to follow along and contribute to the project. Project Milacron is the effort to replace the home grown church management system behind crossroads.net.



The Latest 2

We've identified a promising piece of software called Ministry Platform. We're hopeful in an architecture that has a small core service that performs basic "church management" with many special tools built around that core. We're dedicated to open source. Next steps are to identify areas that need built around Ministry Platform. We're exploring a content management system called Jekyll.

Updated: 2/10/2014

We're working to finalize the scope of the project. We've chosen Ministry Platfrom as our core system. Scope will consist of setting up MP and bulding many modules on top of MP. Next we need to sign a contract with Think Ministry and get an instance installed. We'll then begin to iterate through each module until every feature needed has been delivered.

Updated: 3/20/2014

We've signed a contract with Think Ministry and now have our own install of the Ministry Platform software. Our high level scope has been finalized and a project plan based off major epics (modules) has been outlined. We've kicked off our first sprint of development and finished the first iteration of the new website.

Updated: 4/17/2014

We were moving along pretty good and making good progress in producing functioning code. We've established a solid dev workflow, got continuous deployment in place, and established helpful dev tools(ex. gulp). The API Ministry Platform provided didn't turn out to be what we expected, it's not formatted in JSON like you'd expect but rather the .NET way. Therefore, we've had several debates in how to resolve this problem, for now we've created a prototype API wrapper in node.js. We're currently exploring different architecture options and should have that nailed down in the next couple of weeks.
